Microsoft Fixes 90 New Flaws, Including Actively Exploited NTLM and Task Scheduler Bugs

Posted by Akasha on November 13, 2024

Microsoft's latest Patch Tuesday update addresses 90 security flaws across its platforms, including two actively exploited vulnerabilities in Windows NT LAN Manager (NTLM) and Task Scheduler. The flaws span Critical to Moderate severities, with 52 remote code execution (RCE) issues in total.

Among the actively exploited vulnerabilities:

  • CVE-2024-43451 (CVSS 6.5) - NTLM Hash Disclosure Spoofing
  • CVE-2024-49039 (CVSS 8.8) - Task Scheduler Privilege Escalation

CVE-2024-43451 allows attackers to retrieve a user’s NTLMv2 hash, potentially enabling unauthorized authentication. This vulnerability has seen previous exploitation patterns in NTLM attacks earlier this year. CVE-2024-49039, conversely, involves privilege escalation, enabling attackers to execute restricted RPC functions.

New Vulnerabilities in Edge, Kerberos, and .NET

Additional updates include 31 vulnerabilities in Microsoft Edge and critical vulnerabilities such as:

  • CVE-2024-43498 (CVSS 9.8) - RCE in .NET & Visual Studio
  • CVE-2024-43639 (CVSS 9.8) - RCE in Windows Kerberos

The Azure CycleCloud vulnerability (CVE-2024-43602, CVSS 9.9) ranks as this month's highest-risk flaw, with low-complexity RCE through a simple request to modify configurations.

Broader Industry Updates and CSAF Adoption

In addition to Microsoft, major vendors such as Adobe, Google, and Cisco released critical security patches. Notably, Microsoft has also adopted the Common Security Advisory Framework (CSAF) to enhance vulnerability disclosure transparency.

These updates underscore the ongoing efforts of major tech firms to address vulnerabilities as the complexity of cyber threats continues to grow.
